Coding Standards & Guidelines

Unified rules for writing clean, secure, and maintainable code. These conventions apply to all projects under the Jackal namespace.

Naming conventions identifiers

Consistent naming improves readability and reduces cognitive load. Follow the table below for all new code.

construct style example
variables / functions camelCase userName, calculateTotal()
classes / types / interfaces PascalCase UserProfile, AuthService
constants (immutable) UPPER_SNAKE MAX_RETRY_COUNT, API_BASE_URL
private / internal _camelCase (JS/TS) _privateHelper, _cache
booleans prefix: is, has, should isVisible, hasAccess

// 👍 good · // 👎 avoid
let elapsedTime; // let d;
class AuthGateway; // class auth_gateway;

Formatting & style syntax

Indentation: 2 spaces (no tabs). Max line length: 100 characters. Braces: K&R style (same line).

// JavaScript / TypeScript example function fetchData(url: string): Promise { if (!url) { throw new Error('URL required'); } return fetch(url).then(res => res.json()); } // consistent spacing around operators and after commas
  • 2 spaces
  • no trailing spaces
  • EOF newline
  • no semicolons? — use them (explicit)
Documentation comments

All public APIs, functions, and complex logic must be documented using JSDoc (or equivalent).

/** * Calculates the total price including tax and discount. * @param {number} basePrice - original price (must be positive) * @param {number} taxRate - tax percentage (0–100) * @param {number} [discount=0] - discount percentage * @returns {number} final price rounded to two decimals */ function computePrice(basePrice, taxRate, discount = 0) { ... }

For inline comments, explain why, not what (the code shows what).

Security best practices 🔒 mandatory

All Jackal code must adhere to these non‑negotiable rules:

  • Validate and sanitize all external input (forms, headers, params).
  • Use parameterized queries / ORM to prevent SQL injection.
  • Never hardcode secrets – use environment variables / vault.
  • Apply the principle of least privilege.
  • Set secure HTTP headers (CSP, HSTS, etc.).
// ✅ safe: parameterized query (node-postgres) const result = await db.query('SELECT * FROM users WHERE id = $1', [userId]); // ❌ dangerous: raw string interpolation // const result = await db.query(`SELECT * FROM users WHERE id = ${userId}`);
Error handling & logging resilience

Never swallow exceptions. Use structured logging with levels (debug, info, warn, error).

try { await riskyOperation(); } catch (err) { logger.error({ err, userId, operation: 'riskyOp' }, 'Operation failed'); // rethrow or throw domain-specific error throw new AppError('Unable to complete request', 500); }

Logs must include correlation IDs for distributed tracing.

  • no console.log in production
  • machine‑readable format
Git & commits history

Conventional commits enforced. Structure: type(scope): description

feat(auth): add refresh token rotation fix(api): handle 429 rate limit gracefully docs(readme): update setup instructions refactor(core): split monolith into services

Branch naming: feature/, fix/, chore/ — e.g. feature/jwt-refresh
